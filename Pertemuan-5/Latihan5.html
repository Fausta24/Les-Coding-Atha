Dunia Progamming

Progamming adalah seni dan ilmu untuk mmebuat perangkat lunak. Dalam panduan ini, kita akan menjelajahi konsep, bahasa, dan praktik dalam dunia progamming

1. Dasar-dasar Progamming

  • Algoritma

    • Algoritma adalah langkah-langkah logis untuk menyelesaikan masalah. Algoritma menjadi fondasi setiap progam yang dibuat.

    Pseudocode
    Representasi algoritma menggunakan bahasa manusia atau kode semi-formal.
    Flowchart
    Diagaram visual yang menunjukan alur logika program
  • Struktur Data

    • Struktur data adalah cara menyimpan dan mengorganisir data agar dapat digunakan secara efisien

    • Array

      • Koleksi elemen dengan tipe sama yang diakses melalui indeks.

    • Linked List

      • Rangkaian elemen yang saling terhubung dengan pointer, berguna untuk penyisipan dan pengapusan data yang fleksibel.

    • Stack & Queque

      • Stack mengikuti prinsip LIFO (Last In First Out), sedangkan Queque mengikuti (First In First Out).

2. Bahasa Pemograman

  1. Bahasa Tingkat Tinggi

    2. Bahasa ini mudah dibaca manusia dan bisasanya tidak bergantung pada perangkat keras tertentu

    • Python

      • Cocok untuk pemula, pengembangan web, data science, dan scripting.

    • JavaScript

      • Digunakan untuk pengembangan web interaktif, termasuk fronted dan backend (Node.js).

  2. Bahasa Tingkat Rendah

    3. Bahasa ini lebih dekat dengan bahasa mesin, seperti Assembly atau C, memungkinkan kontrol penuh terhadap hardware.

    • C

      • Bahsa klasik untuk sistem operasi dan embedded progamming.

    • Assembly

      • Bahasa simbolik yang mewakili instruksi mesin secara langsung.

3. Paradigma Pemograman

Object-Oriented Progamming (OOP)
Memodelkan program menggunakan objek dan kelas, memudahkan pemeliharaan dan pengembangan besar.
Functional Progamming
Berfokus pada fungsi murni dan menghindari state yang berubah, populer di Haskell dan Scala.
Procedural Progamming
Memecah program menjadi prosedur atau fungsi, banyak digunakan di C dan Pascal.

4. Praktik Baik dalam Progamming

  • Code Readability

    • Menulis kode yang mudah dibaca dan dimengerti orang lain.

  • Version Control

    • Menggunakan sistem seperti Git untuk melacak perubahan kode dan kolaborasi tim

  • Testing

    • Mengimplementasikan unit test, intregation test, dan functional test untuk memastikan kualitas kode.
